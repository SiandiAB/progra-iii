Reutilización: El Poder de las Librerías

La analogía de LEGO

¿Qué son las librerías? Son bloques de código (como piezas de Lego) que otros programadores expertos ya escribieron y probaron para resolver problemas comunes.

En lugar de fabricar el plástico y moldear la pieza desde cero (lo que equivaldría a meses de ensayo y error programando matemática compleja), usted importa la librería y la "conecta" a su proyecto en un par de minutos de forma segura.

CASO DE ESTUDIO DEL MUNDO REAL

TicketNow: Colapso en la Preventa

Desarrollamos el backend para vender entradas al concierto más esperado del año.

Falla de Robustez

Problema: 100,000 personas hacen click en "Comprar" a la vez.

Un sistema no robusto se apaga y borra las reservas. Uno robusto implementa una Fila Virtual sin crashear.

Poca Flexibilidad

Problema: Solo aceptaban Tarjeta de Crédito. El jefe exige agregar SINPE Móvil mañana mismo.

Si el código es rígido, es imposible; toca reescribir todo el módulo.

Falla de Fluidez

Problema: El servidor falla. Llaman a ingenieros de apoyo de emergencia.

Si el código es espagueti, los nuevos no entenderán nada y no podrán arreglar el problema a tiempo.

No Reutilización

Problema: El equipo decidió programar su propia bóveda de encriptación bancaria desde cero.

Terminaron hackeados. Debieron reutilizar librerías (Stripe, PayPal) certificadas.

Reto: "Mi Primer Repositorio"

GitHub es como un Google Drive para desarrolladores.
El flujo es: PC ➡️ Desktop ➡️ GitHub.com

  1. Crear: En GitHub Desktop, haga clic en File -> New Repository.
  2. Ubicar: En su PC, vaya a la ruta Documentos -> GitHub -> [Su Repo] y cree un archivo Laboratorio0.txt vacío.
  3. Commit 1: Regrese a Desktop, haga su primer Commit (foto al archivo vacío) y presione Publish Repository.
  4. Modificar: Abra el archivo de texto, escriba "Hola, mi nombre es [Su Nombre]" y guarde el archivo.
    Nota: En la pestaña Changes se ven los cambios actuales. Si está vacía, es porque olvidó guardar el archivo.
  5. Commit 2: Haga un segundo Commit con la modificación y presione Push origin para subirlo.
  6. Auditar: Revise la pestaña History. Ahí vive el historial de versiones tanto suyas como de sus compañeros.
  7. Destrucción y Resurrección: Borre la carpeta local de su PC. Luego, use el comando Clone en GitHub para recuperarlo intacto desde la nube.

Introducción a Java y POO

Backend Developer

Es el "cerebro" detrás de la aplicación. Trabajan con Java, Bases de Datos, servidores y APIs, asegurando que la lógica invisible funcione rápido y seguro.

Ejemplo: Construyen la lógica matemática que verifica el saldo y transfiere dinero de su cuenta al banco de la universidad sin que nadie se robe los datos.

Frontend Developer

Son los artistas y arquitectos visuales. Trabajan con HTML, CSS, JavaScript y React para crear todo lo que el usuario ve y toca.

Ejemplo: Diseñan los botones, las animaciones suaves y aseguran que el sistema de matrícula se vea perfecto tanto en una PC gigante como en la pantalla de su celular.

QA / Tester

Los cazadores de errores. Su trabajo es asegurar la calidad del software antes de que llegue al público, protegiendo la reputación de la empresa.

Ejemplo: Intentan "romper" el sistema a propósito ingresando letras donde van números o haciendo 1000 clics por segundo para asegurar que el sistema es a prueba de balas.

Data Science / DB

Dueños de la información. Administran bases de datos de millones de registros, crean estadísticas útiles y programan modelos de Inteligencia Artificial.

Ejemplo: Son los genios matemáticos que diseñan el algoritmo que sabe exactamente qué película recomendarle en Netflix basándose en lo que vio ayer.

Ciberseguridad

Los guardianes de la muralla. Se dedican a buscar vulnerabilidades en el código y diseñar arquitecturas invulnerables contra ataques externos.

Ejemplo: Analizan la plataforma de pagos del concierto de TicketNow para asegurar que los hackers rusos no puedan robarse las tarjetas de crédito clonadas durante el evento.

Cloud / DevOps

Los ingenieros de la nube. Automatizan la subida de código y mantienen los servidores (AWS, Google Cloud, Azure) saludables y escalables.

Ejemplo: Si el sistema de Matrícula colapsa por muchos estudiantes, ellos programan la nube para encender 5 servidores extra automáticamente de forma mágica.

1. Robustez

En palabras simples:

El código es como un tanque blindado. No se "cae" (crashea) ni se cierra abruptamente si el usuario comete un error grave al usarlo o si hay un fallo en la red.

Ejemplo Práctico: En un cajero automático, si usted intenta retirar 10 millones (y no los tiene), el sistema robusto no se apaga lanzando código raro en pantalla, simplemente atrapa el error y muestra un mensaje rojo que dice "Fondos insuficientes", permitiéndole intentar de nuevo.

2. Flexibilidad

En palabras simples:

Capacidad del código de agregar nuevas y complejas funciones mañana sin tener que borrar todo lo que construimos hoy.

Ejemplo Práctico: La aplicación de Uber empezó aceptando solo efectivo. Gracias a tener un diseño flexible y modular, a los 3 meses pudieron agregar pago con Tarjeta y PayPal solo conectando "piezas nuevas", sin tener que reescribir todo el algoritmo de viajes y mapas.

3. Fluidez

En palabras simples:

Código extremadamente limpio, organizado por carpetas lógicas y fácil de leer para otros seres humanos, no solo para la máquina.

Ejemplo Práctico: Si a usted la contratan hoy, entrar a un proyecto ajeno y que las variables se llamen totalFactura y las clases GestorDePagos en lugar de x1 o Archivo3. La fluidez permite que usted entienda la lógica del proyecto en 5 minutos en vez de 5 meses.

4. Portabilidad

En palabras simples:

La independencia técnica. Programar una sola vez y que su código corra igual de bien en computadoras Windows, Mac o servidores Linux.

Ejemplo Práctico: Usted programa el proyecto en su MacBook personal toda la noche. Al llevarlo a la universidad para presentarlo en una PC de laboratorio con Windows 10, funciona mágicamente a la perfección. Ese es el superpoder principal de Java.

5. Reutilización

En palabras simples:

No reinventar la rueda. Usar "Librerías" confiables ya hechas y probadas para resolver tareas genéricas comunes.

Ejemplo Práctico: En lugar de programar la matemática y geometría para dibujar gráficos estadísticos de barras desde cero (tomaría meses), importamos la librería profesional "JFreeChart" y dibujamos un gráfico hermoso en 3 simples líneas de código.

Commit

En palabras simples:

Tomar una "fotografía" o guardar una versión oficial de los cambios en tu código local.

Ejemplo Práctico: Acabas de terminar la pantalla de inicio de sesión. Haces un "Commit" con el mensaje "Login terminado". Si mañana rompes todo el código, puedes volver en el tiempo a esta foto donde todo funcionaba perfecto.

Push Repository

En palabras simples:

Literalmente "Empujar". Es enviar todos tus Commits (fotos locales) a la nube de GitHub para que estén seguros y tu equipo los pueda ver.

Ejemplo Práctico: Trabajaste todo el día en tu computadora sin internet haciendo varios Commits. Al llegar a tu casa, te conectas al Wi-Fi y haces "Push". Tu código viaja a los servidores de GitHub y queda respaldado.

Push Branch

En palabras simples:

Subir a la nube solo una "rama" o versión alternativa del proyecto, sin afectar el código principal (la versión de producción).

Ejemplo Práctico: Estás experimentando con un nuevo diseño oscuro para la app. Creas una rama llamada "diseño-oscuro". Cuando haces "Push Branch", subes a la nube únicamente tus experimentos, permitiendo que tu equipo los revise sin riesgo de romper el diseño actual que usan los clientes.

Clone

En palabras simples:

Es la primera descarga completa de un proyecto que vive en la nube hacia tu computadora.

Ejemplo Práctico: Es tu primer día de trabajo en la empresa. El código de la aplicación ya está en GitHub. Abres GitHub Desktop, pegas el enlace del repositorio y haces "Clone". Automáticamente se descarga toda la historia, carpetas y archivos para que empieces a trabajar.

Fetch

En palabras simples:

Literalmente "Ir a buscar". Es prender el radar de Git para revisar si alguien del equipo subió cambios nuevos a la nube.

Ejemplo Práctico: Usted inicia su turno y ejecuta "Fetch". Git le avisa: "Oye, María subió un archivo nuevo anoche". Lo genial es que no altera ni descarga nada a su computadora todavía, es 100% seguro de ejecutar.

Pull

En palabras simples:

Descargar y aplicar. Trae los cambios detectados en la nube directamente a su computadora local.

Ejemplo Práctico: Después de ver que María subió código útil anoche, usted hace un "Pull". Automáticamente los archivos de su PC se actualizan para incluir el excelente trabajo que ella hizo.

Merge

En palabras simples:

La Fusión. Es el proceso automático donde Git une inteligentemente su código con el de su compañero.

Ejemplo Práctico (Conflicto): Git fusiona archivos sin problema el 99% de las veces. Pero, si usted y su amigo editan la misma línea del mismo archivo al mismo tiempo, Git causa un "Conflicto de Merge" y le pide a un humano que elija manualmente qué versión conservar para no arruinar nada.

Pull Request (PR)

En palabras simples:

Pedir permiso antes de guardar. Usted sube su código y pide que otro humano experto lo revise formalmente.

Ejemplo Práctico: En la vida real, usted no sube su código directo a la rama de Producción (porque si falla, se cae el sistema del banco). Usted abre un PR diciendo: "Líder, terminé la tarea. ¿Puede revisar el código?". Si está correcto, el líder lo aprueba y lo une (Merge).

Java (Core)

¿Qué es? Es un lenguaje de programación multiparadigma, orientado a objetos y sumamente robusto.

Se usa corporativamente porque su Máquina Virtual (JVM) garantiza portabilidad, y su estructura fuertemente tipada obliga a aplicar buenas arquitecturas de diseño, haciéndolo ideal para formar ingenieros.

MySQL 8+

¿Qué es? Un Sistema Gestor de Bases de Datos Relacionales (SGBD) líder en la industria.

Lo usaremos para aplicar la Persistencia de Datos: Que nuestra app de Java no borre sus registros al cerrarse, usando tablas y sentencias SQL.

IntelliJ IDEA

¿Qué es? Un Entorno de Desarrollo Integrado (IDE) creado por JetBrains para maximizar la productividad.

Lo preferimos sobre otros editores porque incluye herramientas para conectarse directamente a la Base de Datos sin instalar otros programas, autocompletado inteligente y refactorización segura.

Git & GitHub

Git es el motor local que toma el historial (fotos) del código. GitHub es la nube donde alojaremos ese repositorio.

Es obligatorio porque nos salvará de catástrofes al permitirnos retroceder en el tiempo, y es la única manera profesional de trabajar los proyectos de grupo de este curso.
